## Q1. 이미지를 모델에 입력하기 전에 리사이징(Resizing)과 정규화(Normalization)를 하는 이유는?

**한 줄 요약:** 리사이징은 모델이 요구하는 고정된 입력 형태를 맞춰주기 위해, 정규화는 픽셀 값의 스케일을 고르게 조정해 학습을 빠르고 안정적으로 만들기 위해 필요하다.

**자세한 설명**

리사이징 (Resizing)
- 딥러닝 모델은 고정된 크기의 입력을 받도록 설계된다. 예를 들어 ResNet 같은 사전 학습 모델은 보통 224x224 입력을 전제로 한다.
- 여러 이미지를 하나의 배치 텐서로 묶어 한 번에 연산하려면 모든 이미지의 크기가 동일해야 한다. 크기가 제각각이면 애초에 하나의 배열로 쌓을 수 없다.

정규화 (Normalization)
- 픽셀 값은 보통 0~255의 정수인데, 이를 [0, 1] 범위로 줄이거나 평균 0 과 표준편차 1이 되도록 조정한다.
- **학습의 안정성과 속도:** 입력 값의 범위가 크거나 특성마다 스케일이 제각각이면 손실 함수의 지형이 찌그러져, 경사하강법이 최적점을 향해 지그재그로 헤매게 된다. 스케일을 고르게 맞추면 수렴이 빨라지고 안정적이다.
- **사전 학습 모델과의 일관성:** ImageNet으로 학습된 모델은 해당 데이터셋의 평균과 표준편차로 정규화된 입력을 전제로 한다. 같은 기준으로 정규화해야 사전 학습된 가중치가 제 성능을 낸다.

정리하면, 리사이징은 **입력의 형태(shape)**를 맞춰주고, 정규화는 **입력의 스케일(scale)**을 맞춰준다.

**예시 / 비유**

- 리사이징은 서류를 제출하기 전에 규격 봉투에 맞춰 A4로 통일하는 것과 같다. 내용이 아무리 좋아도 규격이 안 맞으면 접수 자체가 안 된다.
- 정규화는 여러 나라의 금액을 하나의 통화로 환산하는 것과 같다. 원, 달러, 엔이 뒤섞인 채로 더하면 단위가 큰 쪽이 결과를 지배해버린다.

## Q2. 데이터 증강(Data Augmentation)이란 무엇이며, 이미지에서 주로 쓰는 기법은?

**한 줄 요약:** 데이터 증강은 기존 데이터에 인위적 변형을 가해 학습 데이터를 늘리는 기법으로, 데이터가 부족한 상황에서 과적합을 막고 모델의 일반화 성능을 높이는 것이 목적이다.

**자세한 설명**

왜 하는가
- 딥러닝은 많은 데이터를 필요로 하지만 충분한 양을 확보하기 어려운 경우가 많다.
- 증강을 통해 모델이 특정 패턴(항상 정면을 향한 고양이 등)에만 의존하지 않도록 만들어, 과적합을 방지하고 새로운 데이터에 대한 일반화 능력을 키운다.

주요 기법
1. **회전(Rotation):** 이미지를 일정 각도로 돌려 다양한 방향의 데이터를 학습하게 한다.
2. **뒤집기(Flipping):** 좌우/상하 반전. 특히 수평 뒤집기는 대상이 좌우 대칭성을 가질 때 효과적이다.
3. **잘라내기·크기 변환(Cropping & Scaling):** 일부를 랜덤하게 잘라내기와 크기 변환 / 확대하거나 축소해 대상의 위치나 크기 변화에 견디게 한다.
4. **밝기와 색상 변화(Brightness & Color Jittering):** 밝기, 대비, 색조를 조정해 다양한 조명 환경에서도 잘 작동하게 한다.
5. **노이즈 추가(Adding Noise):** 랜덤 노이즈를 섞어, 깨끗한 데이터뿐 아니라 지저분한 실제 데이터도 잘 인식하게 한다.

주의할 점
- 변형이 레이블의 의미를 훼손하면 안 된다. 예를 들어 숫자 인식에서 6을 180도 회전하면 9가 되어버리므로, 과제에 맞는 증강을 골라야 한다.

**예시 / 비유**

- 한 사람의 얼굴을 여러 각도, 조명, 거리에서 찍어두는 것과 같다. 정면 사진 한 장만 외운 사람은 옆모습을 못 알아보지만, 다양한 모습을 본 사람은 어떤 각도에서도 그 사람을 알아본다.

## Q3. 전이 학습(Transfer Learning)이란 무엇이며, 이미지 분류에서 어떻게 활용하나?

**한 줄 요약:** 전이 학습은 대규모 데이터로 미리 학습된 모델의 가중치를 가져와 새로운 문제에 재사용하는 기법으로, 적은 데이터와 짧은 학습 시간으로도 높은 성능을 얻을 수 있다.

**자세한 설명**

왜 가능한가
- CNN의 **하위층(입력에 가까운 층)**은 엣지, 색, 질감 같은 **범용적인 특징**을 학습한다. 이 특징들은 어떤 이미지 과제에나 공통으로 유용하다.
- 반면 **상위층(출력에 가까운 층)**은 원래 과제에 특화된 고수준 특징을 학습한다.
- 따라서 범용적인 하위층은 재사용하고, 과제 특화적인 상위층만 새 데이터에 맞게 바꾸면 된다.

활용 방법 두 가지
1. **특징 추출(Feature Extraction):** 사전 학습된 합성곱 층 전체를 그대로 얼려(freeze) 특징 추출기로만 쓰고, 마지막 분류 층(Fully Connected Layer)만 새로 교체해 학습한다. 데이터가 아주 적을 때 유리하다.
2. **미세 조정(Fine-Tuning):** **하위층은 고정(freeze)하고, 상위층 일부와 분류 층을 새 데이터에 맞춰 추가로 학습**시킨다. 이때 사전 학습된 가중치가 망가지지 않도록 보통 아주 작은 학습률(learning rate)을 사용한다. 데이터가 어느 정도 있고, 원래 과제와 도메인 차이가 클 때 유리하다.

어디에 쓰이나
- 의료 영상 분석, 소규모 데이터셋 기반의 산업별 특화 모델, 객체 탐지(Object Detection) 등. ImageNet 같은 대규모 데이터셋에서 학습된 모델을 활용해 학습 시간을 줄이고 일반화 성능을 높인다.

**예시 / 비유**

- 이미 영어를 잘하는 사람이 스페인어를 배우는 것과 같다. 알파벳, 문장 구조 같은 기초(하위층)는 이미 갖췄으니 그대로 쓰고, 새 단어와 발음(상위층)만 새로 익히면 된다. 언어를 처음부터 배우는 사람보다 훨씬 빠르다.
