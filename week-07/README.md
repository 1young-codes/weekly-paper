## Q1. Semantic Segmentation이란 무엇이며, 이미지 분류(Classification)와 어떤 차이가 있나요?

**한 줄 요약:** Semantic Segmentation은 이미지의 모든 픽셀에 클래스를 하나씩 할당하는 작업으로, 이미지 전체에 대해 하나의 클래스만 예측하는 이미지 분류와 달리 객체의 위치와 경계를 픽셀 단위로 구분해낸다.

**자세한 설명**

무엇이 다른가
- **이미지 분류:** 입력 이미지 하나에 대해 클래스 하나를 출력한다. 출력은 "이 사진은 고양이다" 같은 라벨 한 개다.
- **Semantic Segmentation:** 입력 이미지와 같은 크기의 지도를 출력한다. 각 픽셀마다 어떤 클래스에 속하는지가 적혀 있다. 즉 "무엇이 있는가"뿐 아니라 "어디에, 어떤 모양으로 있는가"까지 답한다.
- 예를 들어 개와 고양이가 함께 있는 사진을 분류 모델에 넣으면 둘 중 하나의 라벨만 나오지만, Segmentation 모델은 개가 차지한 픽셀 영역과 고양이가 차지한 픽셀 영역을 각각 분리해 보여준다.

같은 클래스의 개별 객체는 구분하지 않는다
- Semantic Segmentation은 클래스 단위로만 나눈다. 개가 두 마리 있어도 둘 다 똑같은 "개" 영역으로 묶이며 1번 개와 2번 개를 구별하지 않는다.
- 개별 객체까지 구분하는 것은 Instance Segmentation이고, 두 가지를 모두 수행하는 것을 Panoptic Segmentation이라고 한다.

어디에 쓰이나
- 자율주행에서 차선과 보행자와 도로 영역 인식, 의료 영상에서 병변 조직 탐지, 위성 영상에서 토지 유형 분류 등. 객체의 경계를 정확히 알아야 하는 작업에서 특히 중요하다.

**예시 / 비유**

- 이미지 분류가 사진에 제목을 다는 일이라면, Semantic Segmentation은 사진 위에 색칠 공부를 하듯 영역마다 색을 칠하는 일이다. 하늘은 파랑, 도로는 회색, 사람은 빨강처럼 모든 픽셀이 어느 한 색을 갖게 된다. 다만 사람이 세 명이어도 셋 다 같은 빨강이라 누가 누구인지는 알 수 없다.

## Q2. Fully Convolutional Networks(FCN)의 주요 특징과 기존 CNN 분류 모델과의 차이점은?

**한 줄 요약:** FCN은 기존 CNN의 완전 연결층을 합성곱층으로 바꾸어 공간 정보를 보존하고, 업샘플링으로 입력과 같은 크기의 예측 맵을 만들어 픽셀 단위 예측을 가능하게 한 모델이다.

**자세한 설명**

핵심 아이디어: 완전 연결층을 합성곱으로 바꾼다
- 기존 CNN은 특징을 추출한 뒤 완전 연결층(Fully Connected Layer)을 거쳐 클래스 하나를 출력한다. 이때 특징 맵을 한 줄로 펴버리기 때문에 **어느 위치의 특징이었는지에 대한 공간 정보가 사라진다.**
- FCN은 이 완전 연결층을 1x1 합성곱으로 대체한다. 그러면 출력이 벡터가 아니라 2차원 맵 형태로 유지되어 위치 정보가 보존된다.
- 덤으로 얻는 이점이 있다. 완전 연결층은 입력 크기가 고정되어야 하지만, 합성곱만으로 이루어진 FCN은 **입력 이미지 크기가 달라져도 동작한다.**

업샘플링으로 해상도를 복원한다
- 합성곱과 풀링을 거치며 특징 맵의 해상도는 계속 작아진다. 이 상태로는 픽셀 단위 예측을 할 수 없다.
- 그래서 전치 합성곱(transposed convolution)으로 다시 키워 입력과 동일한 크기의 예측 맵을 만든다. 흔히 디컨볼루션이라 부르지만 엄밀한 명칭은 전치 합성곱이다.

스킵 연결로 경계를 살린다
- 깊은 층은 의미 정보는 풍부하지만 해상도가 낮아, 그대로 키우면 경계가 뭉개진다.
- 그래서 얕은 층의 세밀한 정보를 가져와 합친다. 얼마나 얕은 층까지 끌어오느냐에 따라 FCN-32s, FCN-16s, FCN-8s로 나뉘며 뒤로 갈수록 경계가 선명해진다.

정리하면 차이점은 세 가지
1. **출력 형태:** 클래스 하나 vs 입력과 같은 크기의 픽셀별 예측 맵
2. **완전 연결층:** 사용함 vs 합성곱으로 대체하여 공간 정보 보존
3. **해상도 복원:** 불필요함 vs 업샘플링과 스킵 연결로 원본 크기까지 복원

**예시 / 비유**

- 기존 CNN이 보고서를 읽고 마지막에 "합격"이라는 도장 하나를 찍는 것이라면, FCN은 보고서 위에 줄마다 형광펜을 칠해 어느 부분이 무엇에 해당하는지 표시해주는 것이다. 도장을 찍는 순간 위치 정보는 사라지지만, 형광펜은 위치를 그대로 남긴다.

## Q3. GAN에서 생성자(Generator)와 판별자(Discriminator)의 역할은?

**한 줄 요약:** GAN은 가짜 데이터를 만들어내는 생성자와 진짜와 가짜를 구별하는 판별자가 서로 속이고 잡아내며 경쟁하는 구조이고, 이 적대적 학습을 통해 생성자가 점점 실제와 구별하기 어려운 데이터를 만들어내게 된다.

**자세한 설명**

생성자 (Generator)
- 랜덤한 노이즈 벡터를 입력받아 실제 데이터와 유사한 데이터를 만들어낸다.
- 목표는 판별자를 속이는 것이다. 즉 판별자가 자기 결과물을 진짜라고 판정하도록 만드는 방향으로 학습한다.

판별자 (Discriminator)
- 입력받은 데이터가 실제 데이터인지 생성자가 만든 가짜인지 구별한다. 본질적으로 진짜와 가짜를 나누는 이진 분류기다.
- 목표는 속지 않는 것이다. 진짜는 진짜로, 가짜는 가짜로 정확히 판정하는 방향으로 학습한다.

적대적 학습이 작동하는 방식
- 두 네트워크의 목표가 정반대이므로, 하나가 잘하게 되면 다른 하나에게는 더 어려운 문제가 주어진다. 이 경쟁이 서로를 밀어 올린다.
- 처음에는 판별자가 가짜를 쉽게 잡아내지만, 생성자가 정교해질수록 판별이 어려워진다. 이상적인 균형점은 판별자가 더 이상 구별하지 못해 정답률이 절반 수준에 머무는 상태다.
- 다만 이 균형을 잡기가 까다로워 학습이 불안정해지기 쉽고, 생성자가 판별자를 속이기 쉬운 몇 가지 결과물만 반복해서 만들어내는 **모드 붕괴(mode collapse)** 문제가 발생하기도 한다.

**예시 / 비유**

- 위조지폐범과 감별사의 관계와 같다. 위조범은 더 그럴듯한 가짜를 만들려 하고, 감별사는 그것을 잡아내려 한다. 서로를 이기려는 과정에서 위조 기술과 감별 기술이 함께 정교해지고, 결국 감별사가 진짜와 구별할 수 없을 정도의 위조지폐가 만들어진다.

## Q4. Diffusion 모델이 이미지 생성에서 어떻게 활용되며, 어떤 장점이 있나요?

**한 줄 요약:** Diffusion 모델은 이미지에 노이즈를 조금씩 더해가는 과정을 거꾸로 되돌리도록 학습해서, 순수한 노이즈로부터 단계적으로 이미지를 복원해내는 생성 모델이다.

**자세한 설명**

동작 원리
- **전방향 과정(Forward Process):** 원본 이미지에 아주 약한 노이즈를 여러 단계에 걸쳐 조금씩 더해, 결국 완전한 무작위 노이즈가 되게 만든다. 이 과정은 정해진 규칙에 따르므로 학습할 것이 없다.
- **역방향 과정(Reverse Process):** 모델은 각 단계에서 **더해진 노이즈가 무엇이었는지를 예측**하도록 학습한다. 학습이 끝나면 순수한 노이즈에서 시작해 노이즈를 조금씩 걷어내는 과정을 반복하며 새로운 이미지를 만들어낸다.

장점
1. **고품질 생성:** 여러 단계에 걸쳐 조금씩 다듬기 때문에 세밀하고 자연스러운 결과를 얻는다. GAN에서 흔한 모드 붕괴 문제도 적어 생성 결과의 다양성이 높다.
2. **안정적인 학습:** GAN은 두 네트워크의 경쟁 균형을 맞춰야 해서 불안정하지만, Diffusion은 **노이즈를 예측하는 단순한 회귀 문제**로 학습한다. 적대적 관계가 없어 학습이 훨씬 안정적이다.
3. **넓은 응용 범위:** 텍스트 기반 이미지 생성, 스타일 변환, 이미지 복원과 초해상도, 의료 영상 복원 등 다양한 조건부 생성 작업에 활용된다.

대가도 있다
- 생성할 때 노이즈 제거를 수십에서 수백 단계 반복해야 하므로, 한 번의 순전파로 끝나는 GAN에 비해 **생성 속도가 느리다.** 이를 줄이기 위한 샘플링 가속 기법이 활발히 연구되고 있다.

**예시 / 비유**

- 잘 만든 모래성 사진에 모래를 한 줌씩 뿌려 완전히 뭉갠 뒤, 그 과정을 거꾸로 되감는 법을 배우는 것과 같다. 모래를 한 줌씩 걷어내는 방법만 정확히 익히면, 나중에는 아무 모래 더미에서 시작해도 성을 만들어낼 수 있다.
