## Q1. 지도 학습과 비지도 학습의 차이는 무엇인가요?

**한 줄 요약:** 지도 학습은 정답(레이블)이 있는 데이터로 입력→출력 관계를 배우는 방식이고, 비지도 학습은 정답 없이 데이터 속 숨은 구조나 패턴을 스스로 찾는 방식이다.

**자세한 설명**

- **지도 학습(Supervised Learning):** 입력 데이터와 그에 대응하는 정답 레이블을 함께 사용해 학습한다. 모델은 주어진 입력에 대해 올바른 출력을 예측하도록 훈련된다. 분류(Classification)와 회귀(Regression)가 대표적이다.
- **비지도 학습(Unsupervised Learning):** 레이블이 없는 데이터만으로 데이터 내부의 구조와 패턴을 찾는다. 군집화(Clustering), 연관 규칙 발견, 차원 축소 등에 쓰인다.

**예시 / 비유**

- 지도 학습: 사진 속 물체에 "고양이/개" 라벨이 붙은 데이터로 분류기를 학습시키는 것.
- 비지도 학습: 라벨 없는 고객 데이터를 분석해 비슷한 소비 성향끼리 그룹으로 묶는 것.
- 비유하자면, 지도 학습은 정답지가 붙은 문제집으로 공부하는 것, 비지도 학습은 정답지 없이 흩어진 카드를 비슷한 것끼리 분류해보는 것과 같다.

## Q2. 손실 함수(loss function)란 무엇이며, 왜 중요한가요?

**한 줄 요약:** 손실 함수는 모델의 예측값과 실제값의 차이를 하나의 숫자로 나타내는 함수이며, 이 값을 최소화하는 방향으로 모델이 학습하기 때문에 학습의 나침반 역할을 한다.

**자세한 설명**

- 예측값과 실제값의 오차를 수치화한다. 값(손실)이 작을수록 잘 예측한 것이다.
- 학습이란 결국 이 손실을 최소화하도록 모델의 파라미터(가중치)를 조정하는 과정이다. 경사하강법(Gradient Descent) 등이 손실 함수의 기울기를 따라 가중치를 업데이트한다.
- 문제 유형에 따라 적절한 손실 함수가 다르다. 회귀에는 MSE·MAE, 분류에는 Cross-Entropy 등이 주로 쓰인다.

**예시 / 비유**

- 양궁에서 화살이 과녁 중심에서 얼마나 벗어났는지를 점수로 매기는 것과 같다. 그 점수(손실)를 줄이려고 자세를 계속 교정하는 과정이 곧 학습이다.

## Q3. 결정 트리의 장점과 단점은 무엇인가요?

**한 줄 요약:** 결정 트리는 if-else 규칙으로 데이터를 나눠 예측하는 모델로, 해석이 직관적이라는 장점이 있지만 과적합과 데이터 변화에 대한 불안정성이라는 단점이 있다.

**자세한 설명**

장점
- **높은 해석력:** 결과를 나무 형태로 시각화해 의사결정 과정(if-else 규칙)을 사람이 쉽게 이해할 수 있다.
- **최소한의 전처리:** 정규화나 표준화 같은 스케일링이 필요 없다.
- **다양한 데이터 타입:** 수치형과 범주형 데이터를 한 모델에서 모두 다룰 수 있다.

단점
- **과적합(Overfitting) 위험:** 트리가 깊어질수록 학습 데이터에만 맞춰져 새로운 데이터에 대한 일반화 성능이 크게 떨어진다.
- **높은 불안정성:** 데이터에 아주 작은 변화가 생겨도 트리의 전체 구조와 결과가 크게 달라질 수 있다.
- **축에 수직인 결정 경계:** 축(Axis)에 수직인 분할만 가능해 사선이나 복잡한 형태의 데이터 분포를 학습하는 데 한계가 있다.

> 이러한 단점을 보완하고 성능을 높이기 위해, 여러 개의 트리를 결합한 앙상블 기법인 랜덤 포레스트(Random Forest)나 XGBoost를 주로 활용한다.

**예시 / 비유**

- 스무고개 게임과 같다. "다리가 4개인가?", "날 수 있나?" 같은 예/아니오 질문(분기)을 순서대로 던져 답(예측)에 도달한다.

## Q4. 모델 학습 시 발생하는 편향과 분산, 그리고 두 개념의 관계

**한 줄 요약:** 편향은 모델이 너무 단순해서 생기는 체계적 오차, 분산은 모델이 너무 복잡해서 데이터에 민감하게 반응하며 생기는 오차이며, 둘은 한쪽을 줄이면 다른 쪽이 커지는 Trade-off 관계에 있다.

**자세한 설명**

- **편향(Bias):** 모델이 지나치게 단순해 데이터의 실제 패턴을 제대로 잡지 못할 때 생기는 오차. 높으면 과소적합(Underfitting)이 된다.
- **분산(Variance):** 모델이 지나치게 복잡해 학습 데이터의 사소한 변동까지 외울 때 생기는 오차. 높으면 과적합(Overfitting)이 된다.
- **관계 (Trade-off):** 모델을 복잡하게 하면 편향은 줄고 분산이 커지며, 단순하게 하면 편향이 커지고 분산이 준다. 총 오차가 최소가 되는 균형점을 찾는 것이 목표다.
  - 고편향·저분산: 너무 단순한 모델 → 복잡한 패턴을 못 잡지만 예측의 일관성은 유지된다.
  - 저편향·고분산: 너무 복잡한 모델 → 학습 데이터에는 매우 정확하지만 새로운 데이터에 흔들린다.

**예시 / 비유**

- 양궁 과녁 비유: 편향은 화살들이 중심에서 한쪽으로 쏠려 빗나간 정도(조준 자체가 틀어짐), 분산은 화살들이 서로 흩어진 정도(자세가 들쭉날쭉). 둘 다 낮아야 좋은 사수다.

## Q5. K-폴드 교차 검증에서 K 값을 선택할 때 고려할 점

**한 줄 요약:** K는 데이터셋 크기, 계산 비용, 편향-분산 균형, 데이터 불균형 여부를 함께 고려해 정하며, 일반적으로 5-폴드나 10-폴드가 무난하게 쓰인다.

**자세한 설명**

- **데이터셋 크기:** 데이터가 충분히 크면 작은 K(예: 5)로도 충분하다. 데이터가 작으면 K를 키워(극단적으로는 LOOCV) 학습에 쓰는 데이터를 최대한 확보한다.
- **계산 비용:** K가 클수록 모델을 K번 학습시켜야 하므로 비용이 증가한다. K가 작으면 비용은 줄지만 평가의 변동성이 커질 수 있다.
- **편향-분산 균형:**
  - K가 **클수록** → 매 학습에 더 많은 데이터를 쓰므로 성능 추정의 **편향이 작아진다.** 다만 폴드 간 학습셋이 많이 겹쳐 추정의 **분산은 커진다.**
  - K가 **작을수록** → 학습 데이터가 적어 성능을 다소 비관적으로 추정(**편향 증가**)하지만, 폴드가 더 독립적이라 추정의 **분산은 작아진다.**
- **데이터 불균형:** 클래스가 불균형하면 각 폴드에 모든 클래스가 충분히 포함되도록 Stratified K-Fold를 사용하는 것이 좋다.

**예시 / 비유**

- 시험 공부에 비유하면, K=10은 자료를 10등분해 9칸으로 공부하고 1칸으로 모의고사를 보는 것을 10번 반복하는 것이다. 칸을 잘게 나눌수록 공부량(학습 데이터)은 늘지만, 모의고사 1회분이 작아져 점수가 들쭉날쭉(분산 증가)해진다.

## Q6. 부스팅의 특징과 대표 부스팅 모델 (Gradient Boosting · XGBoost 외)

**한 줄 요약:** 부스팅은 약한 학습기(weak learner)들을 순차적으로 결합하되, 이전 모델의 오류·잔차를 다음 모델이 집중적으로 보완하도록 가중치를 주며 예측력을 높이는 앙상블 기법이다.

**자세한 설명**

- **핵심 특징:** 순차 학습 + 오류에 가중치 부여(또는 잔차 보완). 모델을 병렬·독립적으로 학습시키는 배깅(Bagging)과 대조적이다.
- **대표 모델**
  1. **AdaBoost (Adaptive Boosting):** 부스팅의 시초. 이전 모델이 틀린 데이터에 가중치를 키우고, 모델 성능에 따라 최종 투표 가중치를 차등 적용한다. 보통 아주 얕은 트리(Decision Stump)를 기본 학습기로 쓴다. 장점은 오답을 집중 개선해 높은 정확도를 기대할 수 있고 과적합에 상대적으로 강한 점, 단점은 이상치·잘못된 라벨에 민감하고 노이즈가 많은 데이터에 취약한 점이다.
  2. **LightGBM (Microsoft):** 학습 속도와 메모리 효율을 극대화한 모델. 트리를 균형 있게 키우는 Level-wise 대신, 손실이 최대가 되는 리프를 분할하는 Leaf-wise 방식을 쓴다. 장점은 매우 빠르고 메모리를 적게 써 대규모 데이터에 강하며 범주형을 자체 처리하는 점, 단점은 데이터가 적으면(약 1만 건 이하) 과적합이 쉽고 하이퍼파라미터 튜닝이 까다로운 점이다.
  3. **CatBoost (Yandex):** 범주형 데이터 처리에 특화. 데이터 누수(Data Leakage)를 막기 위한 정렬 부스팅(Ordered Boosting)을 도입했고 대칭(Symmetric) 트리를 구성한다. 장점은 범주형을 인코딩 없이 입력해도 잘 처리하고 기본 파라미터로도 성능이 우수하며 과적합 방지가 강하고 예측 속도가 빠른 점, 단점은 정렬 부스팅 연산 때문에 훈련 시간이 다른 모델보다 길 수 있는 점이다.

> 정리하면, 대규모 데이터셋과 속도가 중요하면 LightGBM을, 범주형 변수가 많고 튜닝 없이 높은 성능을 원하면 CatBoost를 선택하는 것이 유리하다.

**예시 / 비유**

- 부스팅은 오답노트 학습법과 같다. 한 번 풀고 틀린 문제만 골라 다음에 집중적으로 다시 푸는 것을 반복하며 약점을 메워간다. 반면 배깅은 여러 학생이 각자 따로 풀고 다수결로 답을 정하는 방식이다.

## Q7. 차원 축소 기법 — 주성분 분석(PCA)과 요인 분석(FA)의 차이

**한 줄 요약:** PCA는 데이터의 분산을 최대한 보존하며 차원을 압축하는 데 목적이 있고, FA는 관측된 변수들 뒤에 숨은 잠재 요인을 찾아 변수 간 관계를 해석하는 데 목적이 있다.

**자세한 설명**

- **주성분 분석(PCA):** 데이터의 분산을 가장 잘 설명하는 새로운 축(주성분)을 찾아 그 축을 기준으로 데이터를 변환한다. 변수 간 상관관계는 고려하지만 각 변수의 본래 의미에는 큰 비중을 두지 않는다. 주목적은 데이터의 구조적 압축이다.
- **요인 분석(FA):** 관찰된 여러 변수들이 몇 개의 공통된 잠재 요인에 의해 설명된다는 가정 하에 그 요인을 추정한다. 주목적은 변수 간 관계와 데이터 구조의 해석이다.

**예시 / 비유**

- 학생 성적 데이터를 떠올려보자. 국어·영어·수학 등 여러 과목 점수 뒤에 "학업 능력"이라는 보이지 않는 공통 요인이 있다고 보고 그것을 추출하는 것이 FA다. 반면 PCA는 그 점수들을 분산을 최대한 살리면서 더 적은 수의 새로운 축으로 압축하는 것이다.
