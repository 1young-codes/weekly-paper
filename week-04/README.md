## Q1. PyTorch와 TensorFlow 비교

### 한 줄 요약
PyTorch는 Pythonic한 설계로 연구 및 프로토타이핑에 강하고, TensorFlow는 대규모 배포와 확장성에 강한 딥러닝 프레임워크입니다.

### 자세한 설명
두 프레임워크의 차이는 크게 사용성과 생태계 두 축으로 나눌 수 있습니다.

**사용성**: PyTorch는 동적 계산 그래프 기반의 Pythonic한 설계 덕분에 코드가 직관적이고 간결해서, 실험을 빠르게 반복해야 하는 연구 개발 단계에서 선호됩니다. TensorFlow는 초기에는 정적 그래프 방식이라 코드가 복잡했지만, 2.0부터 Keras와 통합되고 eager execution이 기본이 되면서 사용성이 크게 개선되었습니다.

**생태계와 배포**: TensorFlow는 TF Serving, TF Lite 등으로 대규모 배포와 모바일/엣지 환경 지원에 강점이 있습니다. PyTorch는 연구 커뮤니티에서 압도적으로 선호되며, 최근에는 TorchServe 같은 배포 도구가 나오면서 프로덕션 환경에서의 격차도 줄어들고 있습니다.

### 예시 / 비유
PyTorch는 자유롭게 스케치할 수 있는 스케치북, TensorFlow는 대량 인쇄까지 고려된 출판 시스템에 비유할 수 있습니다. 아이디어를 빠르게 그려보고 수정하는 단계에서는 스케치북(PyTorch)이 편하고, 완성된 결과물을 수많은 사용자에게 안정적으로 전달해야 하는 단계에서는 출판 시스템(TensorFlow)이 유리합니다.

## Q2. PyTorch 텐서와 NumPy 배열의 차이

### 한 줄 요약
텐서는 NumPy 배열과 같은 다차원 배열이지만, GPU 가속 연산과 자동 미분(autograd)을 지원해 딥러닝 학습에 특화된 데이터 구조입니다.

### 자세한 설명
텐서는 `torch.tensor`로 선언하며, 데이터 조작 인터페이스는 NumPy와 매우 유사합니다. 핵심 차이는 두 가지입니다.

**GPU 가속**: 텐서는 `.to(device)` 메서드로 GPU에 올려서 연산할 수 있습니다. 대규모 행렬 연산이 반복되는 딥러닝 학습에서는 이 병렬 연산 능력이 학습 속도를 좌우합니다. NumPy 배열은 CPU 연산만 지원합니다.

**자동 미분(autograd)**: `requires_grad=True`로 설정된 텐서는 자신에게 가해진 연산을 계산 그래프로 기록하고, 역전파 시 손실 함수에 대한 gradient를 자동으로 계산합니다. NumPy에는 이 기능이 없어서, 경사 하강법 기반의 딥러닝 학습에는 부적합합니다.

### 예시 / 비유
NumPy 배열이 일반 노트라면, 텐서는 자신이 거쳐온 계산 과정을 전부 기억하는 스마트 노트입니다. 결과가 틀렸을 때(손실이 클 때) 어느 단계에서 얼마나 잘못됐는지를 거꾸로 추적(역전파)할 수 있고, 필요하면 훨씬 빠른 작업대(GPU) 위로 옮겨서 계산할 수도 있습니다.

## Q3. CNN을 구성하는 각 레이어의 역할

### 한 줄 요약
CNN은 특징을 추출하는 Convolution Layer, 크기를 줄여 일반화를 돕는 Pooling Layer, 추출된 특징으로 최종 판단을 내리는 Fully Connected Layer로 구성됩니다.

### 자세한 설명
**Convolution Layer**: 필터(커널)라는 작은 행렬이 이미지를 스캔하면서 엣지, 텍스처 같은 국소적 특징을 추출합니다. 이미지의 공간적 구조를 보존한 채 특징을 학습할 수 있다는 것이 가장 큰 장점입니다.

**Pooling Layer**: 특징 맵의 크기를 줄여 계산량을 감소시키고 과적합을 방지해 모델의 일반화 능력을 높입니다. Max Pooling은 영역 내 최댓값만 남겨 가장 두드러진 특징을 보존하고, Average Pooling은 평균을 취해 정보를 부드럽게 축약합니다.

**Fully Connected Layer**: 앞 단계에서 추출된 특징들을 모두 연결해 종합적으로 분석하고, 최종 분류나 예측을 수행합니다.

추가로, ReLU 같은 활성화 함수는 모델에 비선형성을 부여하고, Batch Normalization 같은 정규화 레이어는 학습을 안정화하는 역할을 합니다.

### 예시 / 비유
사진 속 고양이를 알아보는 과정에 비유할 수 있습니다. Convolution Layer는 돋보기로 사진 곳곳을 훑으며 "뾰족한 귀", "수염" 같은 단서를 찾는 탐정이고, Pooling Layer는 그 단서들 중 핵심만 남기고 요약하는 조수이며, Fully Connected Layer는 모인 단서를 종합해 "이건 고양이다"라고 최종 결론을 내리는 판사입니다.

## Q4. AutoEncoder의 적용 상황과 인코더/디코더의 개념

### 한 줄 요약
AutoEncoder는 데이터를 압축(인코더)했다가 복원(디코더)하는 과정에서 핵심 특징을 학습하는 비지도 학습 모델로, 차원 축소, 노이즈 제거, 이상치 탐지, 생성 모델링에 적합합니다.

### 자세한 설명
**적용 상황 4가지**
1. **차원 축소**: 고차원 데이터를 저차원 잠재 공간(latent space)으로 효율적으로 표현해 분석 및 시각화에 활용
2. **Denoising**: 잡음이 섞인 입력에서 원본 데이터를 복원하도록 학습시켜 노이즈 제거
3. **이상치 탐지**: 정상 데이터만 학습한 모델은 이상 데이터를 잘 복원하지 못하므로, 복원 오차가 크면 이상치로 판단
4. **생성 모델링**: 잠재 공간을 활용해 새로운 데이터를 생성

**인코더와 디코더**
- **인코더**: 입력 데이터를 압축해 잠재 공간에 표현합니다. 중요한 특징만 추출하고 불필요한 정보를 버리는 역할입니다.
- **디코더**: 잠재 공간의 압축된 표현을 다시 원래 데이터 형태로 복원합니다. 인코더가 학습한 특징을 기반으로 입력과 최대한 유사한 출력을 만드는 데 초점이 있습니다.

**핵심 차이**는 작업의 방향성입니다. 인코더는 축소(고차원 → 저차원), 디코더는 복원(저차원 → 고차원)을 담당하며, 두 과정이 연결되어 입력과 복원 결과의 차이(재구성 오차)를 최소화하도록 함께 학습됩니다.

### 예시 / 비유
책 한 권을 요약했다가 다시 풀어 쓰는 과정과 같습니다. 인코더는 300쪽짜리 책을 1쪽 요약문으로 압축하는 편집자이고, 디코더는 그 요약문만 보고 원래 책 내용을 최대한 비슷하게 다시 써내는 작가입니다. 요약과 복원을 반복하다 보면 "무엇이 이 책의 핵심인가"를 자연스럽게 배우게 됩니다. 이상치 탐지가 가능한 이유도 여기에 있습니다. 소설만 요약해 본 편집자에게 갑자기 수학 교재를 주면 요약도 복원도 엉망이 되는데, 이 "복원 실패"가 곧 이상치의 신호가 됩니다.
